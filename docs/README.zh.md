# 文档索引

这个目录保存 AgentIsland 当前的架构、运行时和集成文档。

## 从这里开始

- [统一 Agent 协议 v1](./unified-agent-protocol.zh.md)
  面向产品内核的主协议，用于 provider 转换层、capability 驱动运行时，以及统一事件/动作模型。
- [多 Agent 架构](./multi-agent-architecture.zh.md)
  当前的分层运行时模型和长期集成方向。
- [运行时可观测性](./runtime-observability.zh.md)
  日志、诊断和 bridge 可观测性控制。
- [OpenCode 集成计划](./opencode-integration-plan.zh.md)
  基于官方文档整理的 OpenCode 接入计划。
- [Agent 扩展指南（中文）](./agent-extension-guide.zh.md)
  新增 provider 时可参考的中文接入指南。
- [Agent 接入 Checklist](./agent-integration-checklist.zh.md)
  新增 provider 时可直接执行的接入清单。
- [设计刷新 Checklist](./design-refresh-checklist.zh.md)
  当前文档、日志、图标和规划收口工作的实施清单。

## 推荐阅读顺序

1. 先读 [统一 Agent 协议 v1](./unified-agent-protocol.zh.md)，理解当前产品主路径和协议形状。
2. 再读 [多 Agent 架构](./multi-agent-architecture.zh.md)，理解分层模型。
3. 再读 [运行时可观测性](./runtime-observability.zh.md)，理解日志和诊断设计。
4. 接入新 provider 时，先读 [Agent 扩展指南（中文）](./agent-extension-guide.zh.md) 理解分层。
5. 真正落地时，按 [Agent 接入 Checklist](./agent-integration-checklist.zh.md) 执行。
6. 评估下一个 provider 时，参考 [OpenCode 集成计划](./opencode-integration-plan.zh.md)。
7. 执行当前收口工作时，参考 [设计刷新 Checklist](./design-refresh-checklist.zh.md)。
